Angular

Fonte: Guida ufficiale Angular

Angular è un framework che semplifica lo sviluppo su più piattaforme grazie alla sua struttura a componenti, che favorisce la riutilizzabilità. A differenza del predecessore AngularJS, Angular si basa su TypeScript.

Per la creazione di un progetto si utilizza (per comodità e semplicità) Angular Cli che tramite una serie di comandi permette di creare facilemente il progetto e le sue componenti. I comandi principali di Angular Cli sono:

Comando Descrizione
ng new NomeProgetto Crea un progetto
ng generate component NomeComponente Crea un componente
ng generate service NomeServizio Crea un servizio
ng generate module app-routing --flat --module=app Crea un modulo per il routing
ng serve Compila il progetto e lo attiva in localhost:4200
ctrl+c Termina il serve

Ogni componente ha il suo css e il suo template html che vengono generati in automatico da Angular Cli come file separati (in alternativa si possono definire dentro al component). Un componente va incluso nelle pagine html dove si vuole renderizzarlo tramite <app-nomeComponente></app-nomeComponente>.

Per utilizzare un campo di un componente si usa l'interpolazione: {{proprietà}}.

Angular permette di integrare la prima validazione dei form nel codice html.

Observable e RxJs

Observable è un tipo per passare messaggi, utile per la gestione degli eventi e per la programmazione asincrona. Gli Observable sono dichiarativi: si definisce una funzione che viene eseguita quando viene invocato il metodo subscribe(). Viene ritornato un oggetto Subscription che può interrompere la ricezione di notifiche con il metodo unsubscribe().

RxJs è una libreria per la programmazione reattiva tramite il tipo Observable. La libreria mette a disposizione funzioni che permettono una creazione più facile di Observable da eventi, timers o promises e operatori che permettono la manipolazione delle collezioni. Il metodo subscribe() è essenziale per l'esecuzione delle funzioni e degli operatori di RxJs e funziona in modo leggermente diverso in base al contesto di utilizzo.

Per convenzione un Observable viene chiamato con il $ alla fine del nome: Osservatore$ : Observable<T>;.
