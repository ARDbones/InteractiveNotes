Variabili

Le variabili possono essere definite in tre modi:

  • var - variabile base
  • let - varibile locale, block scoped
  • const - costante, block scoped, deve essere inizializzata, non può avere lo stesso nome di una variabile o di una funzione; un oggetto costante può essere modificato nei suoi campi (idem per un array)

Valore finchè non è assegnata = undefined Si può controllare se una variabile ha un valore con === undefined In un contesto booleano una variabile undefined assume il valore false, in un contesto numerico assume il valore NaN (not a number). In un contesto booleano una variabile null assume il valore false, in un contesto numerico assume il valore 0.

Variable Hoisting

JS dà la possibilità di riferirsi ad una variabile dichiarata successivamente (solo con var), tuttavia mantiene il valore undefined quando viene usata prima della dichiarazione. Le variabili globali sono proprietà dell'oggetto global window.

Tipi (6 primitivi + object)

  • Boolean
  • null
  • undefined
  • Number
  • String
  • Symbol

Conversione Tipi

JS è tipato dinamicamente: non necessita che venga specificato il tipo e i tipi vengono convertiti durante l'esecuzione in base alle necessità Quando una variabile ha un valore String + Number, il Number viene convertito a String. Se invece vengono usati operatori diversi dal +, la stringa viene convertita in numero. Un numero memorizzato come stringa è possibile convertirlo manualmente con parseInt e parseFloat

Promises

Nuova funzionalità per gestire il flusso di operazioni asincrone. Un oggetto Promise può avere 4 stati:

  • pending - in attesa
  • fulfilled - operazione a buon fine
  • rejected - operazione fallita
  • settled - fulfilled o rejected, ma non pending
